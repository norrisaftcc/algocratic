Git Security Enforcement:

# Never commit secrets (we see everything)
git config --global core.excludesfile ~/.gitignore_global
echo ".env" >> ~/.gitignore_global
echo "*.key" >> ~/.gitignore_global
echo "*_secret*" >> ~/.gitignore_global
echo "config.json" >> ~/.gitignore_global

# Emergency secret removal
git filter-branch --force --index-filter \
  'git rm --cached --ignore-unmatch PATH_TO_SECRET' \
  --prune-empty --tag-name-filter cat -- --all

# Pre-commit security scan
if git diff --cached | grep -E "(password|secret|key|token|api_key)" ; then
  echo "⚠️  SECURITY VIOLATION DETECTED - COMMIT BLOCKED"
  exit 1
fi

Code Review Security Checklist (Mandatory Enforcement):

  • Input validation: Sanitize ALL user inputs (XSS prevention)
  • SQL injection: Use parameterized queries, never string concatenation
  • Authentication: Verify JWT signatures, check token expiration
  • Authorization: Check user permissions on every endpoint
  • HTTPS everywhere: No sensitive data over HTTP (career-ending)
  • Error handling: Don't expose stack traces to users
  • Dependencies: Audit npm/pip packages regularly

When Security Incidents Occur (And They Will):

  1. Contain: Isolate affected systems immediately
  2. Document: Log everything, screenshots, timestamps
  3. Notify: Alert relevant stakeholders (not always management)
  4. Investigate: Root cause analysis, how did they get in?
  5. Remediate: Fix the vulnerability, not just the symptom
  6. Review: Update procedures to prevent recurrence
